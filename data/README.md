Dataset Collection:
For this project, we used the PhiUSIIL Phishing URL Dataset, sourced from the UCI Machine Learning Repository (https://archive.ics.uci.edu/dataset/967/phiusiil+phishing+url+dataset). The dataset contains 235,795 URLs, comprising 134,850 legitimate and 100,945 phishing instances, along with 54 pre-extracted features derived from both the URL string and the corresponding webpage source code. PhiUSIIL was selected as the primary dataset for SafeLink AI because it is recent (published in 2024), large in scale, relatively well-balanced between classes, and its feature set closely aligns with the URL-based features (length, subdomain count, special characters, IP address presence, HTTPS usage, and suspicious keywords) planned for our feature engineering pipeline. The dataset was downloaded as a CSV file and stored in the data/ directory of our project repository for use in preprocessing and model training. Three additional datasets — the Kaggle Malicious Phish dataset, ISCX-URL2016, and a live PhishTank feed — were identified during our literature review as candidates for future cross-dataset validation and multiclass threat categorization, directly addressing the single-dataset evaluation limitation noted across the reviewed literature.
